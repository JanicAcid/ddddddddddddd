# Промт для нового чата — Калькулятор маркировки (Теллур-Интех)

---

## Контекст

У меня есть рабочий сайт **«Калькулятор маркировки»** компании ООО «Теллур-Интех» (сервисный центр кассового оборудования, Санкт-Петербург). Сайт представляет собой **3-шаговый визард** для расчёта стоимости подключения маркировки.

**URL:** https://tellurmarkirovka.vercel.app/

**Репозиторий GitHub:** https://github.com/JanicAcid/ddddddddddddd (ветка `main`)

**Деплой:** Vercel (Hobby Plan, бесплатно). Автоматический деплой при push в main.

## Стек

- **Next.js 15.5.14** + React 19 + TypeScript
- **Tailwind CSS 4** + `tw-animate-css`
- **shadcn/ui** (компоненты: Button, Card, Checkbox, Input, Label, RadioGroup, Separator, Badge)
- **lucide-react** (иконки)
- **ESLint 9** + `eslint-config-next` + `@eslint/eslintrc` (FlatCompat)
- Цветовая схема: navy `#1e3a5f` + gold `#e8a817` + светло-голубой серый градиент фон

## Логика приложения (page.tsx)

### Шаг 1 — Выбор кассы
- RadioGroup: Меркурий, Атол (+ чекбокс «Сигма»), Штрих-М, Пионер, AQSI, Эвотор
- Состояние кассы: Новая / Б/у / Старая
- Для Б/у и Старых (не Эвотор): показываются чекбоксы «Обновление прошивки» и «Лицензия ПО» с ценами из `services.ts`
- Для Эвотор: блок с описанием обязательных приложений из магазина
- Контактные данные: Наименование* , Телефон* , Email* , ИНН, Адрес
- Галочка «У меня есть ЭЦП» — обязательна для перехода дальше

### Шаг 2 — Основные услуги
- Перерегистрация кассы в ФНС (2000₽)
- Полная настройка маркировки под ключ (3500₽)
- Карточка: «ТС ПИоТ оплачивается отдельно на ao-esp.ru»

### Шаг 3 — Дополнительно (2 колонки: left — услуги, right — sticky расчёт)
- Сканер 2D (цены по модели кассы из services.ts)
- Карточки товаров (80/60/40₽ за шт. в зависимости от количества)
- Доп. услуги: Продление ЭЦП (1000₽), Обучение (1500₽/час), Замена ФН (2500₽)
- Для Эвотор: «Помощь с установкой приложений» (1500₽)
- Информация о кассе (модель, зав. номер, номер ФН, примечания)
- Для Эвотор: ЛК данные + «Восстановление доступа к ЛК» (500₽)
- Правая колонка: sticky-карточка «Расчёт стоимости» + кнопка «Готово»

### Экран «Готово» (isDone)
- Заказ-наряд с номером и датой
- Данные клиента, кассы, список услуг с ценами, ИТОГО
- Кнопки «Позвонить» (3 номера: +7(812)465-94-57, +7(812)451-80-18, +7(812)321-06-06)
- Кнопка «Написать в Макс» (ссылка: https://max.ru/+79219324163)
- Email: push@tellur.spb.ru
- Кнопка «Распечатать заказ-наряд» (открывает новое окно с HTML для печати)
- Кнопка «Вернуться к редактированию»

### Подсказки (HintButton)
- Click-to-toggle (НЕ hover — для мобильных)
- Портальное окно с backdrop, закрывается кликом вне
- 4 поля: Что это, Зачем, Когда, Пример
- Используются для: fns_reregistration, marking_setup, scanner_2d, ecp_renewal, training, fn_replacement, product_cards, firmware_update, kkm_license, evotor_restore

### Описания услуг — «продающий» стиль
- Акцент на сложность и ответственность работы
- Перерегистрация ФНС: «Ошибка в заявлении = отказ налоговой = касса не работает»
- Настройка маркировки: «Многоэтапная интеграция 6 разных систем», «Малейшая ошибка = касса не пробьёт чек»

## Важные правила

1. **Мобильная адаптация** — основная аудитория: люди 40+ на iPhone/Android (Yandex Browser). Все подсказки — click-to-toggle, НЕ hover
2. **Никакого AQSI «поддержки из коробки»** — убрано
3. **Никакого обновления до ФФД 1.2** как отдельной услуги — убрано
4. **Помощь с установкой приложений Эвотор** — в доп. услугах (Шаг 3), НЕ в блоке приложений
5. **Логотип** — public/logo.webp (600x492), есть logo@2x.webp для retina. Header: 88x72px, Footer: 56x46px. Убраны `rounded-lg`/`rounded` — логотип не квадратный
6. **package.json** — ЧИСТЫЙ, только нужные зависимости. БЕЗ prisma, next-auth, zustand, recharts и прочего мусора
7. **package-lock.json** — генерировать ТОЛЬКО через `npm`, НЕ через bun
8. **next.config.ts** — БЕЗ `output: "standalone"` (Vercel сам разберётся)
9. **Скрипт build** = `next build` (чистый, без cp -r команд)

## Процесс деплоя при правках

1. Клонировать проект из GitHub: `git clone https://github.com/JanicAcid/ddddddddddddd.git`
2. Внести правки
3. `npm install` (если менялись зависимости)
4. `npx next build` — проверить что билд проходит без ошибок
5. `npm run lint` — проверить ESLint
6. Закоммитить и запушить в main → Vercel автоматически пересоберёт
