# Google Apps Script — Backend kassa-cto.ru

## Инструкция по развертыванию

### 1. Создание проекта

1. Перейдите на [script.google.com](https://script.google.com)
2. Нажмите **«Новый проект»**
3. Удалите весь код из `Code.gs`
4. Скопируйте содержимое файла `google-apps-script.js` в `Code.gs`
5. Нажмите **💾 Сохранить** (Ctrl + S)

### 2. Создание Google Sheet (базы данных)

1. Перейдите на [sheets.google.com](https://sheets.google.com)
2. Создайте новую таблицу (название любое, например `kassa-cto-db`)
3. Скопируйте **ID таблицы** из URL:
   ```
   https://docs.google.com/spreadsheets/d/ЭТО_ID_ТАБЛИЦЫ/edit
   ```
4. В редакторе Apps Script нажмите **📁 → Связать таблицу** → вставьте ID
   — либо добавьте в начало скрипта строку:
   ```js
   var SPREADSHEET_ID = 'ваш_ID_здесь';
   ```
   и замените `SpreadsheetApp.getActiveSpreadsheet()` на:
   ```js
   SpreadsheetApp.openById(SPREADSHEET_ID)
   ```

### 3. Инициализация (setup)

1. В редакторе Apps Script выберите функцию **`setup`** в выпадающем списке сверху
2. Нажмите **▶ Выполнить**
3. При первом запуске потребуется **авторизовать скрипт**:
   - Нажмите «Просмотреть разрешения»
   - Выберите свой Google-аккаунт
   - Нажмите «Дополнительные настройки» → «Перейти на страницу (небезопасно)»
   - Разрешите доступ
4. В журнале выполнения (Execution Log) увидите:
   ```
   ✅ Лист «Users» создан. Администратор: admin / tellur2024
   ✅ Лист «Orders» создан с заголовками.
   🎉 Настройка завершена!
   ```

### 4. Публикация как Web App

1. Нажмите **«Развернуть» → «Новое развертывание»**
2. Тип: **«Веб-приложение»**
3. Настройки:
   - **Описание**: `Backend kassa-cto.ru v1`
   - **Выполнять как**: `Я`
   - **Кто имеет доступ**: `Все`
4. Нажмите **«Развернуть»**
5. **Скопируйте URL веб-приложения** — это ваш API endpoint

### 5. Использование API

Все запросы идут на полученный URL. Примеры:

| Action | Метод | URL | Тело |
|--------|-------|-----|------|
| Капча | GET | `?action=captcha` | — |
| Вход | POST | `?action=login` | `{ login, password, captchaAnswer, captchaToken }` |
| Заказы | GET | `?action=getOrders&token=SESSION` | — |
| Новый заказ | POST | `?action=addOrder` | `{ token?, data: {...} }` |
| Обновить заказ | POST | `?action=updateOrder` | `{ token, rowIndex, updates: {...} }` |

### 6. Данные по умолчанию

- **Логин**: `admin`
- **Пароль**: `tellur2024`

Сменить пароль можно через функцию `changePassword('admin', 'новый_пароль')` в редакторе.

## Доступные action

### `?action=captcha` (GET)
Генерирует математическую капчу. Ответ:
```json
{
  "question": "7 + 12",
  "answer": "19",
  "token": "uuid"
}
```

### `?action=login` (POST)
Авторизация. Тело:
```json
{
  "login": "admin",
  "password": "tellur2024",
  "captchaAnswer": "19",
  "captchaToken": "uuid_из_капчи"
}
```
Ответ: `{ "success": true, "token": "session_uuid" }`

### `?action=getOrders` (GET)
Список всех заказов. Параметр: `?token=session_uuid`
Ответ: `{ "success": true, "orders": [...], "headers": [...], "total": N }`

### `?action=addOrder` (POST)
Добавление нового заказа. Тело:
```json
{
  "token": "session_uuid",
  "data": {
    "Дата/время": "01.01.2025 12:00",
    "Клиент": "Иванов И.И.",
    "Телефон": "+7 999 123-45-67",
    "Статус": "новый"
  }
}
```
Ответ: `{ "success": true, "row": 3 }`

### `?action=updateOrder` (POST)
Обновление заказа (требует авторизации). Тело:
```json
{
  "token": "session_uuid",
  "rowIndex": 3,
  "updates": {
    "Статус": "в работе",
    "Комментарий менеджера": "Передано в ремонт"
  }
}
```
Ответ: `{ "success": true }`

## Диагностика (выполнять в редакторе)

| Функция | Назначение |
|---------|-----------|
| `setup()` | Первичная инициализация таблиц |
| `resetAllSessions()` | Очистить все сессии и капчи |
| `listActiveSessions()` | Список активных сессий |
| `changePassword(login, pass)` | Сменить пароль пользователя |

## Ограничения

- **Квота Google Apps Script**: ~20 000 запросов/день (бесплатный тариф)
- **ScriptProperties**: до 500 КБ данных
- **Размер ответа**: до 50 МБ
- **Время выполнения**: до 6 минут на запрос (Web App — до 30 сек)

При необходимости масштабирования переходите на Google Cloud Functions или отдельный backend-сервер.
